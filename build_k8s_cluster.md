# Kubernetes Cluster Installation Guide (kubeadm + containerd)

## Environment

- OS: Ubuntu
- Kubernetes: v1.30
- Runtime: containerd
- Network Plugin: Calico
- Ingress Controller: NGINX Ingress
- Topology: 3 Control Plane Nodes (3 Masters)

---

# 1. Prepare All Servers

> Execute the following steps on **all 3 servers**.

## 1.1 Configure Hosts

```bash
echo -e "192.168.100.105 k8s-master-1
192.168.100.106 k8s-master-2
192.168.100.107 k8s-master-3" | sudo tee -a /etc/hosts
```

---

## 1.2 Update System

```bash
sudo apt update -y
sudo apt upgrade -y
```

---

## 1.3 Create User

```bash
sudo adduser devops
su - devops
cd /home/devops
```

---

## 1.4 Disable Swap

```bash
sudo swapoff -a
sudo sed -i '/swap.img/s/^/#/' /etc/fstab
```

Verify:

```bash
free -h
```

---

## 1.5 Enable Kernel Modules

Create configuration:

```bash
echo -e "overlay
br_netfilter" | sudo tee /etc/modules-load.d/containerd.conf > /dev/null
```

Load modules:

```bash
sudo modprobe overlay
sudo modprobe br_netfilter
```

---

## 1.6 Configure Kernel Parameters

```bash
echo "net.bridge.bridge-nf-call-ip6tables = 1" | sudo tee -a /etc/sysctl.d/kubernetes.conf
echo "net.bridge.bridge-nf-call-iptables = 1" | sudo tee -a /etc/sysctl.d/kubernetes.conf
echo "net.ipv4.ip_forward = 1" | sudo tee -a /etc/sysctl.d/kubernetes.conf
```

Apply:

```bash
sudo sysctl --system
```

---

# 2. Install containerd

## 2.1 Install Required Packages

```bash
sudo apt install -y \
    curl \
    gnupg2 \
    software-properties-common \
    apt-transport-https \
    ca-certificates
```

---

## 2.2 Add Docker Repository

```bash
curl -fsSL https://download.docker.com/linux/ubuntu/gpg \
| sudo gpg --dearmour \
-o /etc/apt/trusted.gpg.d/docker.gpg
```

```bash
sudo add-apt-repository \
"deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
```

---

## 2.3 Install containerd

```bash
sudo apt update -y
sudo apt install -y containerd.io
```

---

## 2.4 Configure containerd

Generate default config:

```bash
sudo mkdir -p /etc/containerd

containerd config default \
| sudo tee /etc/containerd/config.toml >/dev/null
```

Enable SystemdCgroup:

```bash
sudo sed -i \
's/SystemdCgroup = false/SystemdCgroup = true/g' \
/etc/containerd/config.toml
```

Restart:

```bash
sudo systemctl restart containerd
sudo systemctl enable containerd
```

Verify:

```bash
systemctl status containerd
```

---

# 3. Install Kubernetes Components

## 3.1 Add Kubernetes Repository

```bash
sudo mkdir -p /etc/apt/keyrings
sudo chmod 755 /etc/apt/keyrings
```

```bash
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.30/deb/Release.key \
| sudo gpg --dearmor \
-o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
```

```bash
echo "deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.30/deb/ /" \
| sudo tee /etc/apt/sources.list.d/kubernetes.list
```

---

## 3.2 Install kubeadm, kubelet and kubectl

```bash
sudo apt update -y
```

```bash
sudo apt install -y \
kubelet \
kubeadm \
kubectl
```

Hold version:

```bash
sudo apt-mark hold kubelet kubeadm kubectl
```

---

# 4. Reset Cluster (Optional)

If you need to rebuild the cluster:

```bash
sudo kubeadm reset -f
```

```bash
sudo rm -rf /var/lib/etcd
sudo rm -rf /etc/kubernetes/manifests/*
```

# 5. Initialize Kubernetes Cluster

## Topology

```
k8s-master-1
k8s-master-2
k8s-master-3
```

All nodes are Control Plane nodes.

---

## 5.1 Initialize First Master

Run on:

```
k8s-master-1
```

```bash
sudo kubeadm init \
--control-plane-endpoint "10.24.28.111:6443" \
--upload-certs
```

Configure kubectl:

```bash
mkdir -p $HOME/.kube

sudo cp -i /etc/kubernetes/admin.conf \
$HOME/.kube/config

sudo chown $(id -u):$(id -g) \
$HOME/.kube/config
```

---

## 5.2 Install Calico CNI

```bash
kubectl apply -f \
https://raw.githubusercontent.com/projectcalico/calico/v3.25.0/manifests/calico.yaml
```

---

## 5.3 Join Additional Control Plane Nodes

Run on:

- k8s-master-2
- k8s-master-3

Use the command generated by `kubeadm init`:

```bash
sudo kubeadm join 192.168.100.105:6443 \
--token <TOKEN> \
--discovery-token-ca-cert-hash <SHA256> \
--control-plane \
--certificate-key <CERTIFICATE_KEY>
```

Configure kubectl:

```bash
mkdir -p $HOME/.kube

sudo cp -i /etc/kubernetes/admin.conf \
$HOME/.kube/config

sudo chown $(id -u):$(id -g) \
$HOME/.kube/config
```

Verify:

```bash
kubectl get nodes
```

---

# 6. Install Helm

Run on:

```
k8s-master-1
```

Download:

```bash
wget https://get.helm.sh/helm-v3.16.2-linux-amd64.tar.gz
```

Extract:

```bash
tar xvf helm-v3.16.2-linux-amd64.tar.gz
```

Install:

```bash
sudo mv linux-amd64/helm /usr/bin/
```

Verify:

```bash
helm version
```

---

# 7. Install NGINX Ingress Controller

## 7.1 Add Helm Repository

```bash
helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
```

```bash
helm repo update
```

---

## 7.2 Download Chart

```bash
helm pull ingress-nginx/ingress-nginx
```

```bash
tar -xzf ingress-nginx-4.11.3.tgz
```

---

## 7.3 Modify values.yaml

```bash
vi ingress-nginx/values.yaml
```

Change:

```yaml
controller:
  service:
    type: NodePort

    nodePorts:
      http: 30080
      https: 30443
```

---

## 7.4 Create Namespace

```bash
kubectl create namespace ingress-nginx
```

---

## 7.5 Install Ingress Controller

```bash
helm -n ingress-nginx install ingress-nginx \
-f ingress-nginx/values.yaml \
ingress-nginx
```

---

# 8. Verify Installation

Check nodes:

```bash
kubectl get nodes
```

Check pods:

```bash
kubectl get pods -A
```

Check ingress controller:

```bash
kubectl get pods -n ingress-nginx
```

Check services:

```bash
kubectl get svc -n ingress-nginx
```
